# Global Supply Chain Pulse

External-intelligence dashboard showing where global supply-chain pressure is building, in near-real time, from public data feeds. Built in Python/Streamlit, deployable to Render + Vercel (or any container host), AI-summary ready via the Claude API.

> **Status:** v0.1 starter. Architected for free-tier APIs first; paid feeds drop in as new pipeline modules without UI changes.

---

## What it shows

- **Live world map** - major ports (yellow), strategic chokepoints (orange), recent events colored by category (geopolitical / weather / seismic / freight / commodity).
- **Regional risk scores** - composite 0-100 score per region, with component breakdown.
- **Global activity feed** - severity-ranked stream of recent signals from every source.
- **Commodity & macro trends** - rebased commodity prices + FRED macro series.
- **Executive summary** - deterministic by default; optionally generated by Claude when `ANTHROPIC_API_KEY` + `ENABLE_CLAUDE_SUMMARY=true` are set.

## Data sources (all free tier)

| Source | What you get | Key needed |
|---|---|---|
| GDELT 2.0 DOC | Geopolitical event firehose, tone-scored | No |
| ReliefWeb | Active humanitarian disasters | No |
| USGS GeoJSON | Earthquakes M4.5+ in last 7d | No |
| NOAA NWS | US-area active weather alerts | No (User-Agent only) |
| Open-Meteo | Current weather at major ports | No |
| yfinance | Commodity futures prices | No |
| FRED | Macro & commodity time series | Free, instant |
| AISStream | Real-time AIS vessel positions | Free, instant |
| Anthropic | Claude-generated executive summary | Paid (optional) |

## Architecture

```
                 ┌─────────────────────────────────────────────────────┐
                 │                    PIPELINES                        │
   GDELT ──────► geopolitics ┐                                          │
   ReliefWeb ──► geopolitics │                                          │
   USGS ───────► seismic     │                                          │
   NOAA ───────► weather     │   →  data/signals.json  (snapshot)       │
   Open-Meteo ─► weather     │      data/cache.sqlite (HTTP cache)      │
   yfinance ───► commodities │      data/ais_snapshot.sqlite (vessels)  │
   FRED ───────► macro       │                                          │
   AISStream ──► ports_vessels┘                                         │
                 └─────────────────┬───────────────────────────────────┘
                                   │
                 ┌─────────────────▼───────────────────────────────────┐
                 │                  ANALYTICS                          │
                 │  risk_score.py  → regional 0-100 composites         │
                 │  summarizer.py  → structured brief + optional       │
                 │                   Claude executive summary          │
                 └─────────────────┬───────────────────────────────────┘
                                   │
                 ┌─────────────────▼───────────────────────────────────┐
                 │            STREAMLIT UI (app.py)                    │
                 │   world_map · risk_panel · activity_feed ·          │
                 │   trends · exec_summary                             │
                 └─────────────────────────────────────────────────────┘
```

### Why this shape

- **Every pipeline emits `Signal` records** (canonical schema in `pipelines/base.py`). The risk engine and UI never know which source produced what. Adding a paid feed = one new file.
- **Streamlit reads disk, not APIs.** A cron job (`scripts/refresh_data.py`) re-runs every 15 min and writes `data/signals.json`. Pages load instantly; quotas stay protected.
- **Caching is mandatory on free tier.** SQLite-backed `requests-cache` wraps every external HTTP call. TTLs per source live in `config.py`.
- **AIS uses a side-channel.** Streamlit can't hold a WebSocket open. `scripts/refresh_ais.py` listens for ~60s, dumps positions to SQLite, exits. Streamlit reads the SQLite.

## Local quickstart

```bash
# 1. Clone + env
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# edit .env: at minimum add FRED_API_KEY and AISSTREAM_API_KEY (both free).

# 2. Warm the data cache
python scripts/refresh_data.py
# optionally collect vessel positions:
python scripts/refresh_ais.py

# 3. Run
streamlit run app.py
# open http://localhost:8501
```

**Tests:**

```bash
pytest -q
```

## Risk scoring (transparent heuristic)

Each region's score is a weighted sum of five components, each 0..1:

| Component | Built from | Weight (default) |
|---|---|---|
| Geopolitical intensity | GDELT negative-tone density, ReliefWeb alerts in region | 0.30 |
| Weather alerts        | NOAA + Open-Meteo port-wind/precip exceedances | 0.20 |
| Seismic activity      | USGS M4.5+ in lookback window | 0.10 |
| Commodity volatility  | z-score of 1d returns vs 20d stdev | 0.20 |
| Port congestion proxy | AIS vessel density within 50km of major ports | 0.20 |

Weights live in `config.RISK_WEIGHTS` - tune them. Commodity and macro shocks are global, so they hit every region's score equally; refine this if you build sector→region maps.

**Honest caveat:** this is a **transparent heuristic**, not a validated model. Calibrate it against known disruption events (Suez 2021, COVID lockdowns, hurricane landfalls) once you have ground truth.

## Plugging Claude in

`analytics/summarizer.py` already builds a structured brief. To enable the AI exec-summary:

1. Get a key from <https://console.anthropic.com>.
2. In `.env`:
   ```
   ANTHROPIC_API_KEY=sk-ant-...
   ENABLE_CLAUDE_SUMMARY=true
   ```
3. Restart Streamlit. The "AI executive summary" toggle in the sidebar will activate.

The Claude prompt is constrained to summarize **only** what's in the brief - reduces hallucination, keeps the narrative cited. Summary is cached 30 min by default.

## Deploying

### Stage 1 - Render (Streamlit app + cron)

1. Push this repo to GitHub.
2. In Render: **New → Blueprint → connect repo** (it reads `render.yaml`).
3. Set secrets in the Render dashboard:
   - `AISSTREAM_API_KEY`
   - `FRED_API_KEY`
   - `NOAA_USER_AGENT`
   - `ANTHROPIC_API_KEY` (optional)
4. Render provisions:
   - A **web service** running the Streamlit app
   - A **cron** running `scripts/refresh_data.py` every 15 min
   - A **shared 1GB disk** for the cache + signals snapshot
5. Free tier note: the web service sleeps after 15 min idle. First load after sleep ≈ 30s. Upgrade to Starter ($7/mo) for production.

### Stage 2 - Vercel (landing page on your existing site)

1. `cd landing && vercel --prod`
2. Edit `landing/index.html` and set the iframe `src` to your Render URL (e.g. `https://supply-chain-pulse.onrender.com`).
3. In your domain DNS, point a subdomain (`pulse.yourdomain.com`) to the Vercel deployment.
4. From your main site, link the user to `https://pulse.yourdomain.com`.

### Why this split

Streamlit needs a persistent Python process - Vercel can't host that. Render hosts the live dashboard. Vercel hosts the marketing/landing wrapper + iframes the dashboard. You get the SEO + branding of your existing site and the right runtime for each piece.

## What to add next (suggested order)

1. **Calibrate the risk weights** against 5–10 known historical disruption events.
2. **Persist a signal history** (append-only parquet) so you can chart pressure-over-time.
3. **Add a paid feed** - best ROI is probably a container-rate index (Drewry WCI scraping is free but brittle; Freightos has a paid feed).
4. **Sector view** - pivot from region to "auto / electronics / energy / agri" by mapping commodities + chokepoints to industries.
5. **Email/Slack digest** - fold `build_brief()` into a daily push.
6. **Auth + rate limit** - if the site goes public.
